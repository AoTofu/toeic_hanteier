# TOEIC Reading 判定アプリ

TOEIC Reading の正答データを JSON で読み込み、A/B/C/D の選択肢を押しながらその場で採点できる、単一ファイル構成の静的アプリです。

ビルドは不要で、`index.html` をブラウザで開くだけで動きます。授業用の簡易採点ツール、模試の自己採点、正答データの確認用 UI として使いやすい形を目指しています。

## できること

- 正答データを JSON の貼り付けまたはファイル読み込みで反映
- 問題ごとに A/B/C/D をクリックして回答
- 判定後に正解数、回答数、正答率を集計
- 問題ごとの正誤と正答を一覧で確認
- 問題単位で `choices` を上書きできるので、A/B/C/D 以外の選択肢にも対応

## 使い方

1. `index.html` をブラウザで開きます。
2. 画面上部の JSON 入力欄に正答データを貼り付けるか、JSON ファイルを読み込みます。
3. 各問題の選択肢をクリックして回答します。
4. `判定する` を押すと、正誤と集計結果が表示されます。

## JSON 形式

最低限必要なのは `questions` 配列です。`choices` は全体共通でも、各問題ごとでも指定できます。

```json
{
  "title": "TOEIC Reading Sample",
  "choices": ["A", "B", "C", "D"],
  "questions": [
    { "number": 101, "answer": "A" },
    { "number": 102, "answer": "C" },
    { "number": 103, "answer": "B", "label": "Part 5" },
    { "number": 104, "answer": "D" }
  ]
}
```

各項目の意味は以下の通りです。

- `title`: セット名。読み込み成功メッセージに表示されます
- `choices`: 全問題共通の選択肢
- `questions[].number`: 問題番号
- `questions[].answer`: 正答
- `questions[].label`: 任意のラベル
- `questions[].choices`: その問題だけ別の選択肢を使いたい場合に指定

## ローカルで動かす

依存パッケージはありません。次のどちらかで十分です。

```bash
open index.html
```

または、任意の静的サーバーで配信してください。

## 補足

- 採点や入力処理はすべてブラウザ内で完結します
- 回答データの永続保存機能はありません
- フレームワークを使わず、1 ファイルで扱えることを優先しています
